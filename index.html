About the Proposal

This is based on the chaotic function called a Bifurcation Tree. The idea is, that no matter what the inputs are, you are unable to predict the output. So, if you have an input to the function, say a list of keywords, you input it into the function and receive an output, which maps to a point on the bifurcation tree graph which will act as the private and public key. Theoretically, without knowledge of both of these points, you can't figure out the original input. Knowledge of one will reduce the possible points to a cross-section that if we make large enough, will make it infeasible to brute force. Especially in large applications, needing enough points on the tree to avoid collision is trivial, adding more degrees of precision to the input will allow more points of output on the tree. Plus, running this hashing algorithm will allow for O(1) running time, severely outpacing even the fastest elliptic curve algorithm.

The bifurcation diagram shows the forking of the periods of stable orbits from 1 to 2 to 4 to 8 etc. Each of these bifurcation points is a period-doubling bifurcation. The ratio of the lengths of successive intervals between values of r for which bifurcation occurs converges to the first Feigenbaum constant. Source: bifurcation diagram
